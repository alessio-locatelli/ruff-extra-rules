# Behavioral contract audit: synthesis — General Failure Principle pass (ch. 35)

`docs/behavioral_contract.md` chapter 35 (General Failure Principle) was audited by applying its 10-question rubric to the fix-application pipeline as a whole — `_check_file()`/`_apply_fixes()`/`_refresh_stale_positions()` in `CheckOrchestrator`, `atomic_write_text()`, and `CacheManager` together — rather than to any single check or subsystem, since chapters 1–34 had each already been audited against one narrow, chapter-scoped slice by issues #30–#44. This ticket's own job was to catch whatever fell in the seams between those 15 audits: a decision that was locally correct in one chapter's scope but produces an inconsistent or silently-wrong result once combined with a decision made by a different, chapter-scoped audit. Full findings, the complete chapter-by-chapter disposition, and the rubric walkthrough are in `docs/audits/0016-behavioral-contract-audit-synthesis-general-failure-principle.md`.

## Decision

One cross-cutting gap crossed the bar. `ast_checks/_discovery.py`'s `_list_python_files_in_dir()` (added by issue #33/`docs/adr/0015`, ch. 12/13) expands a directory CLI argument via plain `git ls-files`, which lists only tracked files, while `_prefilter.py`'s `git_grep_filter()` (hardened by issue #43/`docs/adr/0024`, ch. 22/23) deliberately always searches an _explicitly_-named file regardless of its tracked/ignored status. `docs/adr/0024` explicitly judged the resulting directory-scan asymmetry "not the same gap" — reasonable within that audit's own chapter-scoped view, but wrong once the two paths are viewed as one system: naming an untracked file explicitly (`python -m pre_commit_hooks.ast_checks src/new_file.py`) catches its violation, while naming the containing directory instead (`python -m pre_commit_hooks.ast_checks src/`) silently skips the identical file — a false-clean result with no diagnostic, reproduced against the current code (see `docs/audits/0016`). This ADR revises that specific conclusion; `docs/adr/0024` has been amended with a forward pointer to this ADR, so the two records don't stand in unresolved contradiction. The gap itself is filed as [issue #67](https://github.com/alessio-locatelli/ruff-extra-rules/issues/67) rather than fixed inline here, per this ticket's own acceptance criteria.

The audit also found that chapter 33 (Compatibility and Upgrade Behavior) had never been fully dispositioned: issue #32/`docs/adr/0014` scoped itself to "the cache-invalidation-on-version-change portions ... the rest of ch. 33 is CLI/config UX and release process, not caching," and no later ticket picked up the remainder. Walking through the remaining items now, they're all either already satisfied by decisions recorded elsewhere — no configuration file exists at all (`docs/adr/0019`), so the configuration-migration items don't apply; `AGENTS.md`'s own "breaking changes are allowed and expected" is this project's compatibility policy, so a behavior change between releases is permitted by design; serialized cache data already carries its own `version` field, checked and safely discarded on mismatch — or don't apply to a solo, no-external-consumer hobby project with no CI and no formal release process. No code change resulted; this was a gap in the audit trail's own completeness, closed here rather than by a follow-up ticket since there's nothing to implement.

Every other chapter (1–32, 34) already had a recorded real-fix-or-not-applicable disposition from its own prior audit, reconfirmed rather than re-litigated by this pass; see `docs/audits/0016` for the full table and the other cross-cutting interactions checked and ruled out (SIGTERM handling against the cache lock; the production entry point actually used by pre-commit/prek/direct-CLI; `--exclude` ordering against directory expansion; cross-check diagnostic ordering).

## Consequences

- No production code changed by this ticket itself; the one real gap found is tracked as [issue #67](https://github.com/alessio-locatelli/ruff-extra-rules/issues/67).
- Chapter 33's disposition is now complete: the cache-invalidation portion was fixed by `docs/adr/0014`, and the remainder is judged not applicable or already satisfied, recorded in `docs/audits/0016`.
- `docs/audits/0016` is the closing reference for the full 35-chapter behavioral contract audit series (issues #30–#45): every chapter now has a recorded disposition across `docs/adr/0011`–`0027` and their linked `docs/audits/0001`–`0016` reports.
