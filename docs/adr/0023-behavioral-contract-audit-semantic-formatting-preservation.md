# Behavioral contract audit: semantic & formatting preservation across fixers (ch. 2, 21)

`docs/behavioral_contract.md` chapters 2 (Preservation of Source Semantics) and 21 (Formatting and Source Preservation) were audited against every per-check autofix module (`forbid_vars.py`, `excessive_blank_lines.py`, `misplaced_comment.py`, `redundant_assignment/autofix.py`, `validate_function_name/autofix.py`; `redundant_super_init.py` has no autofix and is out of scope, matching `0011`'s identical scoping note). Sequenced after `0011`'s fix-engine write/validation audit, following on from `0011`–`0022`'s prior audits, per issue #39/#30/#36. Full findings, reproductions, and the review history behind each fix are in `docs/audits/0001-behavioral-contract-audit-semantic-formatting-preservation.md`.

## Decision

`forbid_vars`'s autofix must model Python's actual name-binding and evaluation-time scope rules, not perform a single-scope or textual rename. A rename follows a variable into a nested scope's closure reference only when that reference still resolves to the same binding; it must not cross a scope boundary that shadows the name, and it must correctly separate a binding construct's syntax that runs in the _enclosing_ scope (decorators, defaults, non-deferred annotations, a comprehension's first iterable) from the syntax that runs in the scope it introduces. A name reachable only through `global`/`nonlocal`, or an annotation under PEP 563 deferred annotations, cannot be safely rewritten with this codebase's position-based mechanism, so the check now declines to offer a fix for those rather than risk a silent wrong rename. Suggested replacement names are assigned once, after detection finishes, in ascending scope-depth order, so a name is never chosen without visibility into every ancestor and descendant scope closure-following can actually reach — order in the source no longer matters.

Two further, unrelated defects in other fixers were fixed the same way: `misplaced_comment` now reuses each line's own terminator instead of hardcoding `\n`, so a CRLF or CR-only file's untouched lines are no longer silently converted; and `redundant_assignment`'s trailing-comment detection now uses `tokenize` instead of a hand-rolled string scan, which could previously misparse a string containing an escaped quote and delete a real comment along with the assignment it was attached to.

## Consequences

- Autofixes are more conservative in ambiguous cases: a violation reachable only via `global`/`nonlocal`, or (under deferred annotations) an annotation that merely shares a name with an outer local, is reported but not offered as fixable.
- A closure reference is renamed only when it demonstrably resolves to the renamed binding; recognized scope-shadowing constructs now include parameters, `except`/`match` captures, PEP 695 type parameters, imports (including dotted), and `del`, not just plain assignment.
- Suggested names are chosen with visibility across the whole reachable scope tree (ancestors and descendants), preventing cross-scope collisions that a purely local uniqueness check would miss.
- Comment/CRLF handling in `misplaced_comment` and `redundant_assignment` now goes through shared, tokenize-based primitives in `_base.py` (`line_terminator()`, `classify_comment_lines()`, `normalize_for_tokenize()`) rather than ad hoc heuristics, reusable by future fixers.
- A bare, non-`self.`/`cls.` reference to an enclosing function's variable from directly within a class body is still not followed by `forbid_vars`'s rename — a known, pre-existing limitation, unchanged by this audit. Python's class-body scoping is one of its trickiest quirks, and the existing detection-side exclusion (`ForbiddenNameVisitor.visit_ClassDef`) means this is reachable only via a reference, not as a rename source, making it low-value to chase for this project. Revisit only if a concrete case surfaces.
- Everything else audited against ch. 2/21 (string/comment/whitespace preservation, import/exception-order preservation, blank-line/quote-style preservation, diff-minimality, determinism) was already satisfied; see the audit report for the specific citations.

See `docs/audits/0001-behavioral-contract-audit-semantic-formatting-preservation.md` for the full investigation: every reproduced defect, the eight rounds of review that surfaced them, and the regression tests added.
