# redundant_assignment's VariableTracker keeps its own scope model

The original architecture review (ADR 0001) named `VariableTracker` in `redundant_assignment/analysis.py` as a third hand-rolled scope walker duplicating `forbid_vars`/`validate_function_name`'s logic, alongside those two, which were migrated onto the shared `_scope.iter_within_scope`/`SCOPE_NODES` (see the "migrate scope walking" commits). On inspection, `VariableTracker` turned out not to be a duplicate of the same thing: it only opens a new `scope_id` for `FunctionDef`/`AsyncFunctionDef`/`ClassDef` — comprehensions and lambdas are deliberately tracked as a `comprehension_depth` flag on the _enclosing_ scope's assignments/uses, not as their own scope, because the redundant-assignment analysis cares about "is this comprehension result being assigned redundantly," not about modeling comprehension-local variable shadowing precisely. `_scope.py`'s `SCOPE_NODES` includes comprehensions and lambdas as hard boundaries (correct for the rename/collision-avoidance use cases in `forbid_vars` and `validate_function_name`), so routing `VariableTracker` through it would silently change which statements get grouped into which scope for TRI005's own detection logic — a real behavior change to a check the user has confirmed works well in practice and is being enabled by default in this same migration, not a pure refactor.

Also, `VariableTracker` is a single full-file `ast.NodeVisitor` pass that must descend into every scope while tagging `scope_id` as it goes; `iter_within_scope`'s "yield a boundary node but never descend into it" shape doesn't fit that traversal at all — there's no isolated "which node types are boundaries" table to extract from it, since the boundary decision is expressed through separate `visit_FunctionDef`/`visit_ClassDef` method overrides rather than a shared classification.

Decision: leave `VariableTracker`'s scope tracking as its own, narrower implementation. Consolidating it onto `_scope.py` was the wrong abstraction for what it does.
